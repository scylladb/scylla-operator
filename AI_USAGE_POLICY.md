# ScyllaDB Operator AI Usage Policy

This policy sets the rules for using AI in code generation, documentation, and other aspects of the project, and will be enforced for all PRs during review.

## Rules 

1. AI tools can be used for code generation, documentation, and other tasks, including supplementary code review.
2. Every PR must undergo human review.
3. It's the PR author's responsibility to ensure that the AI-generated code or content is adequately tested (particularly: proven by tests to actually solve the problem).
4. It is forbidden to make documentation PRs with untested commands, code snippets, examples and instructions, and hallucinated command outputs.
5. **Reviewers can send the PR back into draft state for self-review if hallucinated content, factual incorrectness, "slop" or other signs of AI use not meeting the quality bar are found in the PR.**
6. Contributors should clearly indicate in their PR description if AI tools were used in the development process, what these tools were, and that they have reviewed and tested the output generated by those tools (see `AI usage` in the PR template).
7. The use of AI tools should not compromise the quality, security, or maintainability of the codebase. All contributions must adhere to our existing coding standards and best practices.
8. Communication on behalf of a human contributor should never be conducted _automatically without human oversight_. This includes, but is not limited to: responding to issues, PRs, or comments. All communication should be reviewed and approved by a human before being sent.
